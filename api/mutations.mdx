---
title: "Mutations"
description: "All available write operations in the NQ GraphQL API."
---

Mutations create, update, or delete data. Like queries, send them as POST requests to `http://localhost:8080`. Most mutations require authentication via the `Authorization` header.

---

## User mutations

### `createUser`

Creates a new user account. In normal app usage, user creation happens automatically on first authenticated request. Use this mutation for programmatic or administrative account creation.

<ParamField path="body.name" type="String" required>
  The user's display name.
</ParamField>

<ParamField path="body.email" type="String" required>
  The user's email address.
</ParamField>

<ParamField path="body.authProvider" type="String">
  The authentication provider identifier (e.g., `"auth0"`).
</ParamField>

<ParamField path="body.authSubject" type="String">
  The provider-issued subject identifier (the `sub` claim from the JWT).
</ParamField>

<ParamField path="body.avatarUrl" type="String">
  URL of the user's avatar image.
</ParamField>

<CodeGroup>

```graphql Example
mutation {
  createUser(input: {
    name: "Alice Kim"
    email: "alice@example.com"
    authProvider: "auth0"
    authSubject: "auth0|abc123"
  }) {
    id
    name
    email
  }
}
```

</CodeGroup>

<ParamField path="body.id" type="UUID" required>

</ParamField>

```graphql Example
    id
    name
    email
    avatarUrl
  }

```

</CodeGroup>

<ParamField path="body.id" type="UUID" required>

## Media mutations

### `createMovie`

Creates a new movie. When `searchDepth` is `0` (or omitted), the backend automatically kicks off a background job to search for related movies via cast, crew, and production company connections.

<ParamField path="body.title" type="String" required>
  The movie title.
</ParamField>

<ParamField path="body.releaseDate" type="Date">
  Release date in `YYYY-MM-DD` format.
</ParamField>

<ParamField path="body.description" type="String">
  Plot summary or description.
</ParamField>

<ParamField path="body.coverUrl" type="String">
  URL of the poster or cover image.
</ParamField>

<ParamField path="body.externalId" type="String">
  TMDB or other external identifier.
</ParamField>

<ParamField path="body.runtime" type="Int">
  Runtime in minutes.
</ParamField>

<ParamField path="body.budget" type="Int">
  Production budget in USD.
</ParamField>

<ParamField path="body.boxOffice" type="Int">
  Box office gross in USD.
</ParamField>

<ParamField path="body.cast" type="String[]">
  List of cast member names.
</ParamField>

<ParamField path="body.crew" type="String[]">
  List of crew member names.
</ParamField>

<ParamField path="body.productionCompanies" type="String[]">
  List of production company names.
</ParamField>

<ParamField path="body.genres" type="String[]">
  List of genre names.
</ParamField>

<ParamField path="body.searchDepth" type="Int">
  Internal depth counter for recursive searches. Omit (or pass `0`) for top-level creation.
</ParamField>

<ParamField path="body.maxConnections" type="Int">
  Maximum number of connected media items to fetch during the background search. Defaults to 25.
</ParamField>

<CodeGroup>

```graphql Example
mutation {
  createMovie(input: {
    title: "Eternal Sunshine of the Spotless Mind"
    releaseDate: "2004-03-19"
    runtime: 108
    genres: ["Drama", "Romance", "Science Fiction"]
    cast: ["Jim Carrey", "Kate Winslet", "Tom Wilkinson"]
    crew: ["Michel Gondry"]
    productionCompanies: ["Focus Features"]
  }) {
    id
    title
    releaseDate
    runtime
  }
}
```

</CodeGroup>

## Activity mutations

### `rateMedia`

Rates a media item on a 0–5 scale. Requires authentication.

<ParamField path="body.mediaId" type="UUID" required>
  The UUID of the media item to rate.
</ParamField>

<ParamField path="body.score" type="Float" required>
  Rating score between 0 and 5.
</ParamField>

<CodeGroup>

```graphql Example
mutation {
  rateMedia(
    mediaId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
    score: 4.5
  ) {
    score
    ratedAt
    media {
      id
      title
      averageRating
    }
  }
}
```

</CodeGroup>

### `createActivity`

Creates a tracking record for a media item — for example, marking that you started watching a show or finished reading a book. Requires authentication. Triggers an asynchronous recommendation rebuild for the user and their friends.

<ParamField path="body.mediaId" type="UUID" required>
  The UUID of the media item.
</ParamField>

<ParamField path="body.statusId" type="Int" required>
  The activity status ID. See [Activity status IDs](/api/user-activity#activity-status-ids).
</ParamField>

<ParamField path="body.rating" type="Float">
  Optional rating score.
</ParamField>

<ParamField path="body.review" type="String">
  Optional review text.
</ParamField>

<ParamField path="body.startedAt" type="DateTime">
  ISO 8601 datetime when you started the media.
</ParamField>

<ParamField path="body.finishedAt" type="DateTime">
  ISO 8601 datetime when you finished the media.
</ParamField>

<CodeGroup>

```graphql Example
mutation {
  createActivity(input: {
    mediaId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
    statusId: 3
    rating: 4.5
    review: "One of the best films I have seen."
    startedAt: "2024-01-10T20:00:00Z"
    finishedAt: "2024-01-10T21:48:00Z"
  }) {
    id
    status {
      id
      name
    }
    rating
    review
    finishedAt
  }
}
```

</CodeGroup>

---

```graphql Example
  }
```

</CodeGroup>

### `acceptFriendRequest`

Accepts an incoming friend request. Returns the new friend's user object. Requires authentication. Triggers an asynchronous recommendation rebuild for both users.

<ParamField path="body.requestID" type="UUID" required>
  The UUID of the friend request to accept.
</ParamField>

<CodeGroup>

```graphql Example

    id
    name
    email
  }
}
```

</CodeGroup>
