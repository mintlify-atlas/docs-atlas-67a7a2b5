---
title: "Quick start"
description: "Get from zero to a running NQ instance and your first recommendation in minutes."
---

This guide walks you from a fresh machine to a working NQ instance — Neo4j running, the Go backend serving GraphQL, and the mobile app open on your device.

<Tip>
  NQ requires both the backend and frontend running simultaneously. Open two terminal windows before you begin: one for the backend and one for the frontend.
</Tip>

---

<Steps>

<Step title="Clone the repository">

Clone NQ from GitHub and move into the project directory:

```bash
git clone https://github.com/grillinr/nq.git && cd nq
```

</Step>

<Step title="Start Neo4j">

NQ ships a `docker-compose.yml` that starts a local Neo4j instance. From the root of the repository, run:

```bash
docker compose -f backend/docker-compose.yml up -d
```

This starts Neo4j with the Bolt port on `7687` and the browser UI at [http://localhost:7474](http://localhost:7474). The default credentials are `neo4j` / `testpass`.

</Step>

<Step title="Configure the backend">

Copy the environment template to create your `.env` file:

```bash
cp backend/.envtemplate backend/.env
```

Open `backend/.env` in a text editor and fill in the Neo4j credentials for the Docker instance you just started:

```bash
NEO4J_URI=neo4j://localhost:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=testpass
NEO4J_DATABASE=neo4j
```

Leave the API key fields blank for now. You can add them later when you connect services like Spotify or Steam.

</Step>

<Step title="Start the backend">

Navigate to the `backend/` directory and start the server:

```bash
cd backend && go run .
```

After a few seconds you should see:

```
connect to http://localhost:8080/ for GraphQL playground
WARNING: TLS is disabled. Set ENABLE_TLS=true for production
```

Open [http://localhost:8080](http://localhost:8080) in your browser. You should see the **GraphQL Playground** — this confirms the backend is connected to Neo4j and ready to accept queries.

</Step>

<Step title="Start the frontend">

Open a **second** terminal window. Navigate to the `frontend/` directory, install dependencies, and start Expo:

```bash
cd frontend && npm install && npx expo start
```

Metro Bundler will start and display a QR code along with an interactive menu.

</Step>

<Step title="Open the app">

Scan the QR code shown in the Expo terminal using the **Expo Go** app on your phone (available free on the App Store and Google Play). Your phone must be on the same Wi-Fi network as your computer.

The app will bundle and open. On subsequent launches the bundle will be faster.

**Alternatives:**
- Press `a` in the Expo terminal to open an Android emulator (requires Android Studio).
- Press `i` to open the iOS Simulator on Mac (requires Xcode).

</Step>

<Step title="Run your first query">

With the backend running, open the GraphQL Playground at [http://localhost:8080](http://localhost:8080) and run the following query to verify your connection end-to-end:

```graphql
query {
  me {
    id
    name
    email
  }
}
```

Click the **Play** button. If you are authenticated, the response will contain your user details. If you see the response without errors, NQ is working correctly.

</Step>

<Step title="Connect a service">

To start seeing personalized recommendations, connect at least one media service by adding its API keys to `backend/.env`. See [Connecting services](/guides/connecting-services) for step-by-step instructions for Spotify, Steam, YouTube, and more.

</Step>

</Steps>
