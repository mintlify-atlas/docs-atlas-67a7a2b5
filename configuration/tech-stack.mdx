NQ is a monorepo containing a Go backend and a React Native (Expo) frontend, connected by a GraphQL API and backed by a Neo4j graph database.

---

## Backend

### Language and framework

| Technology | Version | Role |
|---|---|---|
| Go | 1.25.1+ | Primary backend language |
| gqlgen (`99designs/gqlgen`) | 0.17.78 | GraphQL server and code generator |
| `vektah/gqlparser` | 2.5.30 | GraphQL query parsing |

The backend exposes a single GraphQL endpoint at `/` (port `8080`). All data operations — syncing integrations, fetching recommendations, managing user preferences — go through this endpoint.

### Database

| Technology | Role |
|---|---|
| Neo4j | Graph database for storing media items, users, creators, and their relationships |
| `neo4j/neo4j-go-driver/v5` (5.28.3) | Official Go driver for Neo4j |

Neo4j supports both the cloud-hosted [Aura](https://neo4j.com/cloud/aura/) service and local Docker deployments. See [Database Setup](/configuration/database) for setup instructions.

## Architecture

### Monorepo structure

```
nq/
├── backend/      # Go application (GraphQL API + Neo4j)
├── frontend/     # React Native (Expo) application
├── wiki/         # Project documentation
└── scripts/      # Repository-level utilities
```

### Data flow

1. The **frontend** (React Native) sends GraphQL queries and mutations to the **backend** at `http://localhost:8080`.
2. The **backend** resolves queries by reading from and writing to **Neo4j**.
3. Integration syncs call external APIs (Spotify, Steam, etc.) and write the results into Neo4j nodes and relationships.
4. Recommendation queries traverse the graph to find media items with strong relationships to the user's consumption history.

### Design principles

- **Single GraphQL endpoint** — all client-server communication goes through one API, making it easy to introspect and test via the built-in GraphQL Playground at `http://localhost:8080`.
- **Graph database** — Neo4j is chosen because recommendations are inherently a graph problem: finding paths and patterns through networks of users, media, creators, and tags.
- **Mobile-first** — the primary interface is a React Native app that runs on both iOS and Android via Expo.
- **Graceful degradation** — integrations with missing or invalid credentials are skipped silently; the rest of the system continues to function.
