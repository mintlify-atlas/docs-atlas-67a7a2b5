NQ integrations fall into two categories:

- **User data integrations** — sync your libraries, listening history, and reading lists from platforms you already use.
- **Metadata providers** — enrich media items with additional details such as cast, genres, cover art, and ratings.

## User data integrations

These 7 APIs track media you are actively consuming — saved albums, owned games, saved articles, and more.

  <Card title="Apple Music" icon="music" href="/integrations/music">

  <Card title="YouTube Music" icon="play" href="/integrations/music">

  <Card title="Steam" icon="gamepad-2" href="/integrations/gaming">

  <Card title="Twitch" icon="tv" href="/integrations/gaming">

  <Card title="YouTube" icon="play" href="/integrations/video">

## Metadata providers

These 3 APIs enrich media items with additional details — genres, cast, cover art, ISBN data, and more.

## All integrations at a glance

| Integration   | Category          | Auth method                  | Env vars                                             |
| ------------- | ----------------- | ---------------------------- | ---------------------------------------------------- |
| Spotify       | User data         | OAuth 2.0 client credentials | `SPOTIFY_CLIENT_ID`, `SPOTIFY_CLIENT_SECRET`         |
| Apple Music   | User data         | JWT developer token          | `APPLE_MUSIC_API_KEY`                                |
| YouTube Music | User data         | API key                      | `YOUTUBE_API_KEY`                                    |
| Steam         | User data         | API key + user Steam ID      | `STEAM_API_KEY`                                      |
| Twitch        | User data         | OAuth 2.0 client credentials | `TWITCH_CLIENT_ID`, `TWITCH_CLIENT_SECRET`           |
| YouTube       | User data         | API key                      | `YOUTUBE_API_KEY`                                    |
| Instapaper    | User data         | Basic auth                   | `INSTAPAPER_USERNAME`, `INSTAPAPER_PASSWORD`         |
| IGDB          | Metadata provider | OAuth 2.0 via Twitch         | `TWITCH_CLIENT_ID`, `TWITCH_CLIENT_SECRET`           |
| TMDB          | Metadata provider | Bearer token                 | `TMDB_API_READ_ACCESS_TOKEN` _(add manually)_        |
| Open Library  | Metadata provider | None                         | _(none required)_                                    |

## General setup

All credentials are loaded from a `.env` file in the `backend/` directory.

<Steps>
  <Step title="Copy the env template">
    ```bash
    cd backend
    cp .envtemplate .env
    ```
  </Step>
  <Step title="Fill in your credentials">
    Open `backend/.env` and add values for the integrations you want to enable. You only need to configure the integrations you plan to use — NQ gracefully skips any integration whose credentials are missing.

    ```bash
    # Music
    SPOTIFY_CLIENT_ID=your_value
    SPOTIFY_CLIENT_SECRET=your_value
    APPLE_MUSIC_API_KEY=your_jwt_token
    YOUTUBE_API_KEY=your_value

    # Gaming (TWITCH credentials also power IGDB)
    TWITCH_CLIENT_ID=your_value
    TWITCH_CLIENT_SECRET=your_value
    STEAM_API_KEY=your_value

    # Video (add manually — not in .envtemplate)
    TMDB_API_READ_ACCESS_TOKEN=your_value

    # Articles
    INSTAPAPER_USERNAME=your_value
    INSTAPAPER_PASSWORD=your_value

    # Open Library requires no credentials
    ```
  </Step>
  <Step title="Restart the backend">
    ```bash
    go run .
    ```

    The backend reads `.env` on startup. Restart it after any credential change.
  </Step>
</Steps>

<Note>
  Integrations whose required env vars are absent are silently skipped. The rest of the application continues to work normally — you will only see data for integrations that are fully configured.
</Note>
